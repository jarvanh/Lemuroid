# 更新日志 (Changelog)

本项目的所有重要更新都将记录在此文件中。

## [近期更新]

### 🎮 手柄与输入交互
**提交记录:** `f675a74e` (2026-04-27 11:07:15)
- **极致兼容性提升**: 彻底重构了手柄检测逻辑 (`LemuroidInputDeviceGamePad.kt`)。移除了之前强制要求 Android `hasKeys()` 必须上报 A/B/X/Y 等特定按键的严苛条件。极大地改善了 Google Stadia 手柄、国产杂牌手柄以及由于底层系统原因导致键值上报异常的电视遥控器的开箱即用兼容性。
- **智能按键映射提示**: 在游戏启动界面 (`TVGameActivity.kt`) 增加了实时的 UI 状态检测。如果手柄已连接但未能正确映射其默认按键（或者处于被禁用状态），系统现在会触发专用的提示消息，引导用户前往“外接设备”设置中手动重新绑定按键。
- **默认快捷键优化**: 将呼出游戏菜单的默认组合键优先级修改为 `L1+R1`，这对于没有摇杆按压功能的电视手柄来说，是一种更加自然和可靠的后备方案。

### 📺 Android TV 存储与体验适配
**提交记录:** `58618ee3` (2026-04-27 09:57:09), `71dd99bd` (2026-04-27 09:57:09)
- **ROM 目录选择修复**: 彻底解决了在部分 Android TV 上选择 ROM 目录时出现的“点击闪退”以及选中后显示“无 (None)”的问题。
- **路径解析容错机制**: 更新了 `SettingsScreen.kt` 和 `TVSettingsFragment.kt` 的逻辑，现在能够优雅地兼容处理 SAF URI 格式 (`content://`) 和传统的绝对路径格式 (`/storage/emulated/0/...`)。当 SAF 解析失败时，系统会安全地降级为直接显示标准字符串路径。
- **存储访问降级适配**: 暂时将 `targetSdkVersion` 降级至 29，以绕过那些魔改电视盒子上严格且不完整的 Scoped Storage（分区存储）限制，确保自带的传统文件夹选择器 (`TVFolderPickerLauncher`) 能够成功读取和索引游戏文件。
- **UI 体验增强**: 在电视端的设置 UI 中显式地增加了一个 ROM 目录展示项 (`pref_key_tv_rom_directory`)，允许用户直观地看到自己当前选中的扫描路径。

### 🛠 重构与杂项清理
**提交记录:** `f675a74e` (2026-04-27 11:07:15)
- **后台操作监控优化**: 将后台的资源库操作监控 (`PendingOperationsMonitor`) 严格限制为只监听“库索引 (LIBRARY_INDEX)”这一项，防止其他无关的后台操作（如存档同步）在后台默默阻塞 ROM 扫描任务。
- **冗余代码清理**: 移除了此前为了在没有 ADB 环境下强制抓取日志而临时添加的远程日志抓取脚本 (`RemoteLogger.kt`, `log_server.py`) 及其相关的启动逻辑。
